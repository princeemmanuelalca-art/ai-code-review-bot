#  AI Code Review Bot

An automated code review system using GitHub Actions, PyGithub, and Groq's LLaMA 3.3 70B model.

## Features

- **Automated PR Reviews**: Triggers on every pull request (opened/updated)
- **Comprehensive Analysis**: Checks for bugs, security issues, performance problems, and style violations
- **Smart Chunking**: Handles large diffs by splitting into manageable chunks (max 4000 tokens)
- **Structured Output**: Posts organized reviews with severity levels (High/Medium/Low)
- **Synthesis**: Combines multi-chunk reviews into a single cohesive comment

## Setup

### 1. Add Secrets

Go to **Settings → Secrets and variables → Actions** and add:

- `GROQ_API_KEY`: Your Groq API key ([get one here](https://console.groq.com/keys))
- `GITHUB_TOKEN`: Automatically provided by GitHub Actions

### 2. File Structure

```
.github/
└── workflows/
    └── review.yml      # GitHub Actions workflow
review.py               # Main review logic
requirements.txt        # Python dependencies
README.md              # This file
```

### 3. How It Works

1. **Trigger**: Workflow runs on `pull_request` events (`opened`, `synchronize`)
2. **Fetch Diff**: Uses PyGithub to retrieve the PR diff
3. **Chunk**: Splits large diffs into ≤4000 token chunks (by file, then by hunk)
4. **Review**: Sends each chunk to Groq API with a specialized system prompt
5. **Synthesize**: If multiple chunks, combines reviews into one structured comment
6. **Post**: Creates a PR comment with sections:
   - Summary
   - Issues Found (High/Medium/Low severity)
   - Suggestions

## Configuration

### Environment Variables

| Variable | Description |
|----------|-------------|
| `GROQ_API_KEY` | Groq API authentication |
| `GITHUB_TOKEN` | Auto-provided GitHub token |
| `PR_NUMBER` | Target PR number (auto-set) |
| `REPO_NAME` | Repository name (auto-set) |

### Customization

Edit `SYSTEM_PROMPT` in `review.py` to adjust review criteria or output format.

## Example Output

```markdown
##  AI Code Review

## Summary
This PR introduces a new authentication middleware and updates user validation logic. Overall good structure but has critical security concerns.

## Issues Found

### High Severity
- **[Security]**: Hardcoded API key in `config.py:15`
  - **Impact**: Credentials exposed in version control
  - **Fix**: Use environment variables or secret management

### Medium Severity
- **[Performance]**: O(n²) loop in `search_users()`
  - **Impact**: Slow queries with large datasets
  - **Fix**: Use database indexing or hash-based lookup

### Low Severity
- **[Style]**: Missing docstrings in `auth.py`
  - **Impact**: Reduced maintainability
  - **Fix**: Add Google-style docstrings

## Suggestions
- Add unit tests for the new middleware
- Consider using a typed dictionary for config

---
*Generated by Groq (llama-3.3-70b-versatile) via GitHub Actions*
```
